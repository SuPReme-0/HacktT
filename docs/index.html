CORE ARCHITECTURE

🧠

Hardware-Aware Logic

Features a dynamic toggle. Run strictly Active Mode (Qwen 3.5 4B reasoning) for low-spec machines, or unlock Passive Mode to engage full background monitoring if your rig has 6GB+ Dedicated VRAM. Automatic VRAM detection prevents OOM crashes.

👁️

Florence-2 Vision OCR

Passive mode triggers advanced screen capture pipelines. HackT natively reads your IDE and browser windows via Florence-2 Base, detecting phishing attempts, vulnerable code syntax, and suspicious UI elements in real-time. Model loads ephemeral to save VRAM.

🔐

Local Cryptographic Vault

Powered by LanceDB vector search and KùzuDB graph databases. HackT learns from your projects, remembers past vulnerabilities, and builds a personalized, air-gapped threat intelligence database. All data encrypted at rest.

🎙️

Voice Command Interface

Faster-Whisper Tiny runs locally on CPU for real-time speech-to-text. Issue commands hands-free while coding. Piper TTS provides natural voice responses. All audio processing stays on-device—no cloud APIs.

IDE & Browser Integration

WebSocket listeners on ports 8081 (IDE) and 8082 (Browser) enable real-time code stream analysis and phishing detection. VS Code and Chrome extensions available. Passive mode only.

FREQUENTLY ASKED

Is HackT completely offline? +
Yes. After initial installation and model downloads, HackT operates 100% offline. The only cloud dependency is Supabase Auth for user authentication (email + profile name only). All AI inference, threat detection, and vault storage occur locally on your hardware.
What happens if I have less than 6GB VRAM? +
HackT automatically detects your VRAM at startup. If you have less than 6GB dedicated VRAM, Passive Mode will be locked. You can still use Active Mode, which runs Qwen 3.5 4B for chat and RAG queries without Florence-2 Vision or background monitoring.
Does HackT send my code to the cloud? +
Absolutely not. Your code, conversations, and threat intelligence never leave your machine. The only data transmitted to our servers is your email and profile name for authentication via Google SSO. All AI processing happens locally via Qwen 3.5 and Florence-2.
How do I install the VS Code / Chrome extensions? +
After installing HackT Runtime, open the Settings panel in the dashboard. Click "Install" next to IDE Integration or Browser Integration. This will redirect you to the respective extension marketplace. Once installed, the extensions will automatically connect to HackT on ports 8081 and 8082.
Can I customize the AI models? +
Advanced users can replace the default Qwen 3.5 4B GGUF model with any compatible llama.cpp model. Place your custom model in the `models/` folder and update the path in the Python backend configuration. Note: VRAM requirements may vary with different models.
